# Lessons Learned

A running record of non-obvious gotchas encountered while working on
**bestehorn-llmmanager**, and how they were resolved. Read this before non-trivial work
and append to it after resolving anything surprising. See
`.claude/rules/use-lessons-learned.md`.

Entry format:

```markdown
## YYYY-MM-DD — <short title>
- **Context**: what you were doing
- **Lesson**: what was non-obvious / what went wrong
- **Action**: the fix, and how to avoid it next time
```

---

## 2026-06-17 — Claude Code configuration added alongside Kiro
- **Context**: Migrated the project for Claude Code use while keeping it fully usable
  with Kiro (`.kiro/` left untouched).
- **Lesson**: This project deliberately uses the **black/isort/flake8/mypy/bandit**
  toolchain at **line-length 100** (not ruff/120), and
  `src/bestehorn_llmmanager/_version.py` is auto-generated by setuptools-scm and must be
  excluded from every quality tool. The generic Claude Code setup templates assume
  ruff@120 — those defaults were overridden to match the project's existing standards.
- **Action**: The Claude Code rules in `.claude/rules/`, the `.pre-commit-config.yaml`,
  the slash commands, and the hooks all encode black/isort/flake8 @ 100 with the
  `_version.py` exclusion. Do not "upgrade" them to ruff. The `/run-ci` command mirrors
  `.github/workflows/ci.yml`.
