# Lessons Learned

A running record of non-obvious gotchas encountered while working on
**bestehorn-llmmanager**, and how they were resolved. Read this before non-trivial work
and append to it after resolving anything surprising. See
`.claude/rules/use-lessons-learned.md`.

Entry format:

```markdown
## YYYY-MM-DD — <short title>
- **Context**: what you were doing
- **Lesson**: what was non-obvious / what went wrong
- **Action**: the fix, and how to avoid it next time
```

---

## 2026-06-17 — Claude Code configuration added alongside Kiro
- **Context**: Migrated the project for Claude Code use while keeping it fully usable
  with Kiro (`.kiro/` left untouched).
- **Lesson**: This project uses the **ruff (ruff format + ruff check)/mypy** toolchain
  at **line-length 100** (not the 120 default), and
  `src/bestehorn_llmmanager/_version.py` is auto-generated by setuptools-scm and must be
  excluded from every quality tool. The generic Claude Code setup templates assume
  line-length 120 — those defaults were overridden to match the project's standards.
- **Action**: The Claude Code rules in `.claude/rules/`, the `.pre-commit-config.yaml`,
  the slash commands, and the hooks all encode ruff (ruff format + ruff check)/mypy @
  line-length 100 with the `_version.py` exclusion. Keep the line length at 100. The
  `/run-ci` command mirrors `.github/workflows/ci.yml`.
- **Note (2026-06-18, issue #15)**: This entry originally described a
  **black/isort/flake8/mypy/bandit** toolchain. That toolchain was migrated to **ruff**
  (`ruff format` + `ruff check`, with ruff's `S` rules replacing bandit) under issue
  #15; the tool names above have been updated to reflect the migration. The underlying
  lessons (line-length 100, exclude `_version.py` everywhere) are unchanged.
