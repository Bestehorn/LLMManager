# Project Engineering Standards (Claude Code)

This is **bestehorn-llmmanager** — a Python library for AWS Bedrock Converse API
management (multi-model fallback, multi-region failover, fluent MessageBuilder,
parallel processing, retry logic, streaming, prompt caching).

This project is configured for **both Kiro and Claude Code**. The Kiro config under
`.kiro/` is the source of record for Kiro; this `CLAUDE.md` plus `.claude/rules/` is
the Claude Code equivalent. Detailed rules live in `.claude/rules/`. The
non-negotiable essentials:

## Environment
- ALWAYS work inside the project venv (`venv/`). On Windows the activation is
  `venv\Scripts\activate`; prepend it to python/pip commands with `&`
  (e.g. `venv\Scripts\activate & pytest test/`). Recreate the venv if missing/broken;
  never `pip install` without it. See `.claude/rules/use-venv.md`.
- NEVER set shell environment variables (AWS_PROFILE, AWS_REGION, GITHUB_TOKEN, …).
  Pass profile/region explicitly; read credentials from `config/` and `credentials/`.
  See `.claude/rules/no-environment-vars.md`. (Enforced by a PreToolUse hook.)

## Coding standards (no exceptions)
- Comprehensive type hints. JSON field access via string constants, not literals.
- One class per file with matching filename; relative imports within `src/`.
- Named parameters at all call sites: `f(a=5, b="abc")`.
- Errors are exceptions (custom exceptions take a `details=None` kwarg); return `None`
  for *expected* failures only.
- `logging` not `print`; `pathlib.Path` not `os.path`.
- **Line length 100** (project-specific — NOT 120).
- `src/bestehorn_llmmanager/_version.py` is auto-generated by setuptools-scm — NEVER
  modify it and ALWAYS exclude it from black/isort/flake8/mypy/bandit.
- Full detail: `.claude/rules/coding-standards.md`, `.claude/rules/design-principles.md`,
  `.claude/rules/file-organization.md`, `.claude/rules/tech-stack.md`.

## Evidence and honesty
- Every claim to the user is backed by command output, file contents, test results, or
  a documentation lookup. No hedge words for actual behavior.
  See `.claude/rules/no-guessing.md`.

## Tests
- Tests must pass. Never skip/xfail/delete a test to go green; fix the root cause.
  See `.claude/rules/tests-must-not-fail.md` and `.claude/rules/always-test-e2e.md`.
  Coverage target 80% (`--cov=bestehorn_llmmanager`). Unit tests mirror `src/` under
  `test/bestehorn_llmmanager/`; AWS integration tests under `test/integration/`.

## Before / after any task
- Pre-work: read `docs/forLLMConsumption.md`, `docs/ProjectStructure.md`, `README.md`.
  See `.claude/rules/pre-work.md`.
- Post-activity: run pytest + black + isort + flake8 + mypy + bandit (all in venv,
  `_version.py` excluded); update docs; clean `tmp/`. See `.claude/rules/post-activity.md`.

## Documentation MCP servers
- Before using AWS/Bedrock/Strands/AgentCore/CDK APIs, consult the corresponding MCP
  server (configured in `.mcp.json`). See `.claude/rules/use-doc-mcp-servers.md`.

## Git / remote
- All GitHub API operations go through `scripts/github_wrapper.py` (not `gh`/`glab`/raw
  curl). See `.claude/rules/use-git-wrapper-scripts.md`. Remote CI must pass before a
  change is considered done — see `.claude/rules/remote-ci-must-pass.md`.

## Read complete command output; never truncate with tail/head/Select-Object.
  See `.claude/rules/no-output-shortening.md`.

## Decision log
- All agents follow `.claude/rules/agent-state-convention.md` for state directories and
  the append-only `DL-NNN` decision log.
